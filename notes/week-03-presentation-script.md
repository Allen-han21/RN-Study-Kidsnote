# AI 활용 사례 발표 멘트
> 개발자를 위한 AI 도입 인사이트

---

## 🎤 오프닝 (30초)

안녕하세요. 오늘은 실제 기업들이 AI를 어떻게 활용해서 개발 생산성을 높이고 있는지 공유하려고 합니다.

자료에는 10개 기업 사례가 정리되어 있는데요, 시간 관계상 **개발자로서 가장 임팩트 있었던 핵심 인사이트** 몇 가지만 집중적으로 말씀드리겠습니다.

---

## 📊 핵심 인사이트 1: 숫자로 보는 AI의 실제 효과 (1분)

먼저, AI 도구가 실제로 얼마나 효과가 있는지 숫자로 보겠습니다.

**GitHub와 MIT의 공동 연구 결과**를 보면:
- Copilot을 사용한 개발자들이 같은 작업을 **55.8% 더 빠르게** 완료했습니다
- Microsoft 내부 데이터는 더 인상적인데요, 반복 작업에서 **71% 시간을 절약**했고, 작업 품질도 **68% 향상**되었습니다

근데 여기서 중요한 건, "AI가 코드를 대신 작성해준다"가 아니라는 겁니다.
Microsoft의 경우 현재 **코드의 20-30%를 AI가 작성**하고 있다고 하는데,
이건 개발자가 더 높은 수준의 문제 해결에 집중할 수 있게 된다는 의미입니다.

## 🚀 핵심 인사이트 2: 토스의 실전 활용 - "2일을 2시간으로" (2분)

SK플래닛과 토스 사례가 특히 인상적이었는데요.

### SK플래닛의 극적인 시간 단축
SK플래닛 개발팀이 **데이터베이스 주석 작업**을 했습니다.
- 테이블 약 200개, 컬럼 약 5,000개
- 원래 **2일** 걸릴 작업이었는데
- AI 활용해서 **2시간** 만에 완료

**95% 시간 단축**이에요. 거의 하루 반을 아낀 거죠.

이게 우리한테 시사하는 건 뭘까요?
우리도 문서화 안 된 테이블이나 레거시 코드에 주석 다는 작업 있잖아요.
이런 **반복적이지만 중요한 작업**에 AI를 활용하면 엄청난 시간을 절약할 수 있다는 겁니다.

### 토스의 테스트 자동화 접근법
토스가 더 재밌는 게, 단순히 코드만 생성하는 게 아니라 **개발 인프라 자체를 AI 친화적으로** 만들고 있어요.

**1) 테스트 자동화**
- 대출 같은 외부 연동 테스트를 위한 **Mock 서버**를 직접 개발
- 외부 API 의존성 없이 안정적인 테스트 환경 구축
- 이게 왜 중요하냐면, 외부 API 장애나 속도 문제 때문에 테스트가 깨지는 일이 없어진다는 거죠

**2) ML 파이프라인 자동화 - Trainkit**
- Feature Store랑 학습 파이프라인을 완전 자동화
- 신분증 검증 서비스를 ML로 만들어서 **94% 정확도** 달성
- 수작업 검증에서 자동화로 전환

**3) API 연동 자동화**
- 사내 MCP 서버 개발
- 반복적인 API 연동 작업을 자동화

토스의 핵심은 뭘까요?
**"AI를 쓴다"가 아니라 "AI가 잘 작동할 수 있는 인프라를 만든다"**는 거예요.
테스트 환경, 파이프라인, API 연동까지 전부 자동화해서,
개발자가 비즈니스 로직에만 집중할 수 있게 만드는 겁니다.

## 🎯 핵심 인사이트 3: 당근마켓 - "작은 것의 큰 효과" (1분)

당근마켓 사례는 규모는 작지만 **실용성**이 인상적이었습니다.

**상품 카테고리 자동 분류**
- 사용자가 중고거래 글 올릴 때, 제목만 보고 AI가 카테고리를 자동 분류
- 겉보기엔 단순해 보이지만, 효과는 3가지:
  1. **사용자 경험**: 판매자가 카테고리 고민 안 해도 됨
  2. **검색 품질**: 정확한 카테고리로 검색 정확도 향상
  3. **운영 효율**: 잘못된 카테고리 수정 작업 감소

이게 주는 교훈이 뭐냐면,
AI 도입이 꼭 거창할 필요 없다는 겁니다.

**"우리 서비스에서 사용자나 운영팀이 반복적으로 하는 분류/태깅 작업이 뭐가 있을까?"**
이런 작은 지점부터 시작해도 충분히 효과를 볼 수 있어요.

예를 들어:
- 고객 문의 자동 분류
- 버그 리포트 우선순위 자동 태깅
- 코드 리뷰에서 변경 유형 자동 감지

이런 것들도 당근마켓처럼 작지만 임팩트 있는 개선이 될 수 있습니다.

## 💡 우리가 할 수 있는 것 (1분)

그럼 우리는 뭘 할 수 있을까요?

자료 마지막에 **"우리 회사라면?"** 질문들을 각 사례마다 정리해뒀는데요,
제가 생각하는 가장 실용적인 접근법 3단계는:

### 1단계: 작게 시작하기 (1주)
- GitHub Copilot이나 Cursor 같은 도구 **무료 체험판**부터 써보기
- 팀 내 관심 있는 개발자 2-3명이 1주일만 써보고 피드백 공유
- "이거 어디에 쓰니까 좋더라" 케이스 발굴

### 2단계: 측정 가능한 파일럿 (2-4주)
- 명확한 목표 설정: 예) "주석 없는 레거시 함수 100개에 문서화"
- 시간 측정: AI 쓰기 전 vs 후
- 품질 측정: 코드 리뷰 통과율, 버그 발생률

### 3단계: 인프라 개선 고려 (1-3개월)
- 토스처럼 테스트 자동화 환경 개선
- CI/CD 파이프라인에 AI 코드 리뷰 추가
- 반복 작업 자동화 스크립트 작성

**핵심은: "완벽한 계획" 보다 "빠른 실험"입니다.**

## 🎬 마무리 (1분)

마지막으로 정리하자면:

**AI 도구는 이제 "써볼까 말까"가 아니라 "언제, 어떻게 쓸까"의 문제입니다.**

숫자가 증명하고 있어요:
- 55-71% 시간 절약
- SK플래닛 77% 현업 활용률
- 2024년 67억 달러에서 2030년 257억 달러로 시장 성장

근데 더 중요한 건,
**토스처럼 인프라를 개선하고, 당근마켓처럼 작은 것부터 시작하는 자세**입니다.

오늘 자료에는:
- 10개 기업 사례와 링크
- 각 사례별 "우리 회사라면?" 브레인스토밍 질문
- 4단계 적용 체크리스트

이 모든 게 정리되어 있으니까,
오늘 세션 끝나고 **팀으로 돌아가서 "우리는 뭘 해볼까?" 대화를 시작하는 게 첫걸음**이 될 것 같습니다.

**질문이나 의견 있으신 분?**

## 📌 Q&A 대비 예상 질문

### Q1: "비용이 얼마나 드나요?"
**A**: GitHub Copilot 기준 개인은 월 $10, 기업은 월 $19입니다.
만약 개발자가 한 달에 하루라도 절약한다면, 시급으로 계산했을 때 충분히 ROI가 나옵니다.
무료 체험판으로 먼저 효과를 측정해보시는 걸 추천드립니다.

### Q2: "보안은 괜찮나요? 코드가 외부로 유출되진 않나요?"
**A**: GitHub Copilot Business는 코드를 학습에 사용하지 않습니다.
민감한 코드는 .gitignore처럼 Copilot 제외 설정도 가능하고요.
토스 같은 금융권에서도 쓰고 있다는 건, 보안 검토를 통과했다는 의미입니다.

### Q3: "AI가 잘못된 코드를 생성하면 어떡하나요?"
**A**: 그래서 "코드 리뷰"가 더 중요해졌습니다.
AI는 보조 도구고, 최종 책임은 개발자에게 있습니다.
오히려 Microsoft 연구에서 코드 품질이 68% 향상됐다는 건,
AI가 베스트 프랙티스를 제안해서 품질이 올라간다는 의미기도 합니다.

### Q4: "주니어 개발자 성장에 방해되지 않나요?"
**A**: 오히려 반대일 수 있습니다.
- 반복 작업 줄이고 → 비즈니스 로직 이해에 집중
- AI 제안을 보면서 → 베스트 프랙티스 학습
- 빠른 프로토타입 → 더 많은 시도와 학습 기회

단, 맹목적으로 받아들이지 않고 "왜 이렇게 제안했을까?" 생각하는 게 중요합니다.

### Q5: "우리 레거시 코드베이스에도 적용 가능한가요?"
**A**: SK플래닛과 Amazon 사례가 바로 그겁니다.
- SK플래닛: 5,000개 컬럼 주석 자동화
- Amazon Q: 레거시 Java → 최신 버전 자동 변환

오히려 레거시일수록 AI의 도움이 더 클 수 있습니다.
문서화, 리팩토링, 테스트 작성 같은 "하고 싶었지만 시간이 없었던 일"들을 할 수 있게 되니까요.
